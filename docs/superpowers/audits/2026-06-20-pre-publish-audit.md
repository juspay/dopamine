## Executive Summary

risk is **not** a secret leak â€” git history is clean of credentials (only `.env.example` was ever
committed). The dominant risk is **795 committed files of scraped third-party Instagram data**

### B2 â€” No `files` allowlist / no `private: true` â€” npm publish ships everything
`npm pack --dry-run` = 1043 files / 23.4 MB incl. all scraped data, `.env.example`,
`node_modules_broken/`, both lockfiles. Only guard is `npmPublish:false` in `.releaserc.json`,
which a plain `npm publish` ignores.
**Fix:** add to `package.json`:
```json
"private": true,
"license": "MIT",
"description": "AI video retrieval and processing pipeline powered by Gemini",
"author": "Sachin Sharma",
"engines": { "node": ">=20" },
"files": ["dist", "README.md", "LICENSE"]
```
`"private": true` is the structural guard â€” it makes `npm publish` error unconditionally.

### B3 â€” No LICENSE file
No `LICENSE`, no `license` field; `README.md:326` says "Private project. All rights reserved." while
`@semantic-release/npm` is wired in.
**Fix:** create `LICENSE` (MIT, or `UNLICENSED` if staying closed), add `"license"` to `package.json`,
replace README line 326. If data isn't removed first, no OSS license can legitimately cover it.

### B4 â€” `.env.example` commits real personal/internal values
`INSTAGRAM_USERNAME=your_instagram_username`, `VERTEX_PROJECT=your-gcp-project-id`, `GOOGLE_CLOUD_PROJECT_ID=your-gcp-project-id`,
`MODEL=gemini-3.1-flash-image-preview` (non-GA id).
**Fix:** replace with `your_instagram_username` / `your-gcp-project-id` / `gemini-2.0-flash`.
