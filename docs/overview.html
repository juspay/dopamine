How To Use

Simple commands to run the full pipeline, serve the dashboard, or trigger on-demand.

$ npm run pipeline
Run the full 11-step pipeline: download, classify, extract knowledge, generate dashboard
$ npm run serve
Serve the interactive dashboard at http://localhost:3001 with search, filter, and sort
$ npm run start
Start the webhook server on port 3000 + daily 3am cron scheduler
$ python3 download_videos.py
Quick video sync -- download new saved videos from Instagram without full pipeline
$ curl -X POST :3000/trigger
Trigger a pipeline run via webhook (returns immediately, runs in background)
$ curl :3000/status
Check if the pipeline is currently running

Architecture

Design Principles

▼
Sequential Pipeline
Steps have hard data dependencies. No queue-based multi-agent complexity for a single-user batch workload.
Resume Mode
Every AI step checks existing state before processing. Crash-safe: state saved after each successful item.
Exponential Backoff
2s delay between requests. Up to 5 retries with exponential backoff on 429/RESOURCE_EXHAUSTED errors.
Large File Fallback
Videos over 50MB automatically use thumbnail JPEG for AI analysis instead of full video binary.

Zod Schemas

▼
ClassificationSchema
category, subcategory, tags[], description, language, mood
KnowledgeSchema
transcript, visual_description, links_and_resources[{url, description, timestamp}], key_takeaways[], topics[]
LinksSchema
links[{name, url, type (shown_on_screen | mentioned_verbally | inferred_from_context), description, timestamp}]

Gemini Constraints

▼
disableTools: true is required when using schema + output.format:"json" with Gemini
Cannot use tools (websearchGrounding) AND JSON schema in the same call
Avoid z.union() in schemas -- triggers "Too many states" on Gemini
Video files must be absolute paths in input.files
LinkResolver uses no schema -- Google Search grounding cannot combine with JSON output
