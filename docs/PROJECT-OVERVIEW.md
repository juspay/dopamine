# Dopamine -- Instagram Saved Videos Knowledge Engine

**A personal knowledge management system that turns Instagram saved videos into a searchable, organized knowledge base.**

Built by **Sachin Sharma** ([@user](https://instagram.com/user)) at **Juspay**

---

## What It Is

Dopamine is an end-to-end pipeline that takes your Instagram saved videos and transforms them into structured, searchable knowledge. It downloads every saved video, uses AI to classify and understand each one, extracts transcripts and links, resolves those links to real URLs, and generates a beautiful interactive dashboard -- all fully automated with daily cron scheduling.

The system processes **235 videos** across **21 categories**, producing **115 deep knowledge base entries** with full transcripts, **479 extracted and resolved links**, and **116 markdown files** -- all generated by **Gemini 2.0 Flash** on Google Cloud Vertex AI.

### Step-by-Step Breakdown

| Step | Agent | What It Does | AI? |
|------|-------|-------------|-----|
| 1 | **MetadataAgent** | Login to Instagram via instagrapi, fetch all saved posts, write `metadata.json` | No |
| 2 | **DownloadAgent** | Download new MP4s for video-type saved posts, track `known_pks.json` for instant sync | No |
| 3 | **PropertiesAgent** | Run ffprobe on each MP4 (duration, resolution, FPS, codec), extract thumbnails with ffmpeg | No |
| 4 | **ClassifierAgent** | Send each video to Gemini 2.0 Flash with Zod schema -- returns category, subcategory, tags, description, language, mood | **Gemini** |
| 5 | **KnowledgeAgent** | For AI & Tech videos: extract full transcript, visual descriptions, key takeaways, topics | **Gemini** |
| 6 | **LinkExtractAgent** | Re-watch all knowledge base videos, extract every URL shown on screen or mentioned verbally | **Gemini** |
| 7 | **LinkResolverAgent** | Use Gemini + Google Search grounding to resolve product/tool names to real HTTPS URLs | **Gemini** |
| 8 | **CatalogAgent** | Join all JSON state files into unified `catalog.json` + `catalog.csv` | No |
| 9 | **OrganizerAgent** | Create symlink-based folder structure under `videos/classified/{category}/` | No |
| 10 | **MarkdownAgent** | Generate per-video markdown files + `INDEX.md` under `knowledge_base/` | No |
| 11 | **DashboardAgent** | Build self-contained `dashboard/index.html` with embedded catalog data | No |

## Architecture

### Design Principles

1. **Single-process sequential pipeline** -- Steps have hard sequential data dependencies; queue-based multi-agent adds complexity with no benefit for a single-user batch workload.
2. **NeuroLink as AI library** -- One `NeuroLink` instance created in `runner.ts`, passed to all AI agents. `generate()` is the unit of composition.
3. **Resume mode on all AI steps** -- Every agent checks if each item exists in output JSON before processing. State is saved after each successful item.
4. **Rate limiting with exponential backoff** -- 2s delay between requests, up to 5 retries with exponential backoff on 429/RESOURCE_EXHAUSTED errors.
5. **Large file fallback** -- Videos >50MB use thumbnail JPEG instead of full video for Gemini calls.
6. **known_pks.json for instant sync** -- Previously downloaded video primary keys are tracked, so sync only downloads new saves.

### AI Agent Pattern

All four AI agents (Classifier, Knowledge, LinkExtract, LinkResolver) follow the same canonical structure:

1. Load existing state (resume mode)
2. Iterate over items, skip already-processed
3. Call `neurolink.generate()` with appropriate schema
4. Parse response with Zod
5. Save state after each successful item
6. Rate limit between requests

### Key Constraints

| Constraint | Reason |
|---|---|
| `disableTools: true` with Zod schemas | Gemini rejects tools + JSON schema in the same call |
| No `z.union()` in schemas | Triggers "Too many states" error on Gemini |
| Absolute paths for video files | Required by NeuroLink's video analysis |
| LinkResolver uses no schema | Google Search grounding (tool) cannot combine with JSON schema output |

## Infrastructure

### Servers

| Server | Port | Purpose |
|--------|------|---------|
| Webhook | 3000 | `POST /trigger` runs pipeline, `GET /status` checks if running |
| Dashboard | 3001 | Serves interactive video dashboard with search, filter, sort |

### Scheduling

- **Daily cron at 3:00 AM** -- Automatic pipeline run to sync new saved videos
- **Webhook trigger** -- On-demand pipeline execution via HTTP POST
- **Mutex lock** -- Prevents concurrent pipeline runs

### Resilience

- **Resume mode** -- Every AI step skips already-processed items
- **Exponential backoff** -- Rate limit handling with up to 5 retries
- **Large file fallback** -- Videos >50MB use thumbnails for AI analysis
- **State persistence** -- JSON state saved after each successful item processing
